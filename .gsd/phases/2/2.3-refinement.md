# Plan 2.3: Interaction Refinements

## Objective
Address user feedback regarding Sphere rotation (lack of drag interaction) and particle sensitivity (too weak).

## Context
- src/components/Scene.jsx
- src/components/Sphere/ParticleSphere.jsx

## Tasks

<task type="auto">
  <name>Enable Drag Rotation</name>
  <files>src/components/Scene.jsx</files>
  <action>
    1. Import `OrbitControls` from `@react-three/drei`.
    2. Add `<OrbitControls enableZoom={false} enablePan={false} autoRotate={true} autoRotateSpeed={0.5} />` to the Canvas.
    3. Remove the manual `rotation.y` logic in `ParticleSphere.jsx` if it conflicts, OR keep `OrbitControls` `autoRotate` as the primary rotation method.
    4. Ensure `OrbitControls` allows manual override (drag).
  </action>
  <verify>Manual check: Dragging mouse rotates sphere. Sphere rotates automatically when idle.</verify>
  <done>OrbitControls active.</done>
</task>

<task type="auto">
  <name>Boost Particle Sensitivity</name>
  <files>src/components/Sphere/ParticleSphere.jsx</files>
  <action>
    1. In `ParticleSphere.jsx`:
       - Increase `threshold` (interaction radius) from `1.5` to `2.0` or `2.5`.
       - Increase `lerp` factor (force) from `0.2` to `0.8` (or use a dynamic force based on distance squared).
       - Remove manual `rotation.y` in `useFrame` to let `OrbitControls` handle global rotation, OR keep it if it's a separate local rotation (usually better to let OrbitControls handle camera/scene rotation). *Decision: Remove manual mesh rotation to avoid conflict with OrbitControls user feel.*
  </action>
  <verify>Manual check: Particles react strongly to cursor.</verify>
  <done>Physics updated.</done>
</task>

## Success Criteria
- [ ] User can rotate sphere with mouse drag.
