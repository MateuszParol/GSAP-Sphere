# Plan 3.1: Point System & Markers

## Objective
Implement the data structure and visual markers for the 4 Points of Interest on the sphere.

## Context
- SPEC Goal 2: 4 distinct, clickable points.
- Tech Stack: React Three Fiber, @react-three/drei (`Html` component).
- Aesthetics: "Cyber / Sci-Fi", pastel colors.

## Tasks

<task type="auto">
  <name>Define Point Data</name>
  <files>src/data/points.js</files>
  <action>
    1. Create `src/data/points.js`.
    2. Export an array `points` containing 4 objects:
       - `id`: unique string.
       - `position`: Vector3 (x, y, z) - normalized to surface (radius ~1.8).
       - `label`: string (e.g., "About", "Works", "Socials", "Contact").
       - `color`: string (pastel hex).
       - `type`: 'link' | 'modal' (for Contact).
  </action>
  <verify>Check file existence.</verify>
  <done>Data defined.</done>
</task>

<task type="auto">
  <name>Create Marker Component</name>
  <files>src/components/Navigation/PointMarker.jsx</files>
  <action>
    1. Create `src/components/Navigation/PointMarker.jsx`.
    2. Use `Html` from `@react-three/drei` to render a DOM element.
    3. Style:
       - Small circle/dot or neon square.
       - Label visible on hover or always.
       - "Occlude" prop on `Html` to hide when behind sphere.
    4. Animations:
       - Pulse effect using CSS or GSAP.
  </action>
  <verify>Visual check.</verify>
  <done>Component created.</done>
</task>

<task type="auto">
  <name>Render Points in Scene</name>
  <files>src/components/Navigation/PointsGroup.jsx, src/components/Scene.jsx</files>
  <action>
    1. Create `src/components/Navigation/PointsGroup.jsx` to map over `points` data and render `PointMarker`s.
    2. Add `PointsGroup` inside the `<Canvas>` in `Scene.jsx` (inside the rotating group if points should stick to surface).
    3. *Crucial Decision*: Do points rotate WITH the sphere? Yes. So they must be children of the rotating `ParticleSphere` or a shared rotating Group.
    4. Refactor `Scene.jsx` to have a `<group>` that contains both `ParticleSphere` and `PointsGroup`, and this group is controlled by `OrbitControls` (auto-rotate).
  </action>
  <verify>Browser check: Points appear on sphere surface and rotate with it.</verify>
  <done>Points rendered.</done>
</task>

## Success Criteria
- [ ] 4 Markers visible on the sphere.
- [ ] Markers rotate attached to the sphere surface.
- [ ] Markers are occluded when behind the sphere.
